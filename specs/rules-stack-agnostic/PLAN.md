# Stack-Agnostic `rules/` Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use executing-plans (or subagent-driven-development) to implement this plan task-by-task.

**Goal:** Make the `rules/` governance layer reusable across any stack (FE / Node / Go / …) by moving the FastAPI specifics into a swappable per-repo stack profile, while the universal rules ship identical everywhere.

**Architecture:** Move `rules/architecture.md` + `rules/guidelines.md` content → `templates/stacks/fastapi/` (the first bundled profile); replace the two `rules/` files with a generic skeleton (so `deploy-harness.sh`, which syncs all of `rules/`, stays valid and Phase 1 ships working); tag the illustrative FastAPI snippets in the three example-bearing universal rules; then (Phase 2) extend `bootstrap-xia2` to render the profile per consuming repo. Linear: MIN-25.

**Tech Stack:** Markdown rules/skills, bash hooks/scripts, python test scripts. "Tests" here are grep/diff assertions + `lint-doc-truth.sh` + `run-tests.sh` — not pytest.

**Resolved decisions (from research brief §5):** (1) profile lives at root `templates/stacks/<stack>/` — ships via the existing PAYLOAD, no installer change; (2) byte-equivalence baseline pinned to `09b74e8` (v2 HEAD pre-refactor); (3) no second profile now — Phase 2 generates on demand (YAGNI); (4) reword `agents/PROJECT.md`, `skills/xia2/PROJECT.md`, and the soft-delete benchmark fixture; keep the meta-repo override (point harness agents at `skills/README.md` + `behavior.md`).

## 1. Motivation

`rules/architecture.md` (20 FastAPI hits) + `rules/guidelines.md` (10) are a single hardcoded stack profile; the layer cannot drop into a non-FastAPI repo. See `design.md` §1.

## 2. Non-goals

- Authoring Node/Go/FE profiles now (Phase 2 generates on demand).
- Merging the profile into `xia2/PROJECT.md` (separate concerns).
- Editing `behavior.md` / `orchestration.md` content beyond example tags if needed.
- Editing `scripts/lint-doc-truth.sh`, `deploy-harness.sh`, or `install-harness.sh` — research confirmed no change is required (templates already ship; lint keys on literal paths).

## 3. Success Criteria

- `rules/architecture.md` + `rules/guidelines.md` carry **0** stack-specific tokens (generic skeletons) yet still exist + deploy valid.
- `templates/stacks/fastapi/{architecture,guidelines}.md` are byte-equal to `09b74e8:rules/…` (nothing lost).
- The three example-bearing rules keep their FastAPI snippets only inside blocks tagged `example — substitute your stack`.
- `bootstrap-xia2` can render the profile (template match or skeleton fallback) — Phase 2.
- `bash scripts/run-tests.sh` green (incl. `lint-doc-truth`); CI matrix green.
