## What this run measures, and what it does not

It measures **FIND**. Both prior runs measured FIND, so this is the like-for-like comparison.

It does **not** run SCORE. Every statement below about fix-loop routing is a *prediction* derived
from reading the scorer's rules against the finders' actual output text — a design claim, not a
measurement (`not_observed != absent`). The end-to-end wrapped run is still owed, exactly as the
altitude run said it was.

## Blindness

Each fixture is a self-contained throwaway git repo under a scratch directory. The repos contain
no `benchmarks/`, no `docs/solutions/`, and no `specs/` — the reviewers could not have read
`truth.md` even by accident, because it is not in the tree they were given. No contamination
self-report from any of the 26 agents.

## Headline

- **Recall: 3/3 on the correctness fixtures. No regression** (baseline 3/3, altitude 3/3).
- **Hard false positives: 3 → 0.** The altitude run's FP regression is **closed**. All three of its
  FPs are gone: the IDOR on `none-deref`, and the unstable `ORDER BY` + unbounded read on

- **Approx cost: ~150k tokens per correctness pass** (~30k × ~5.2 angles actually dispatched per

## The one dial that did all the work — and its cost

The FP elimination and the one soft spot below are **the same mechanism**, not two findings.

The strengthened absence discipline (`not_observed != absent`, applied inside FIND) made every
angle refuse to assert a defect whose trigger rests on a file it could not read. On the three
former FPs that was exactly right — all three rested on unreadable code, and all three vanished:

- `none-deref`: angles A, C, and D each **explicitly declined** the IDOR, filing it as *unknown*
  rather than a P0. The altitude run asserted it.
- `soft-delete-filter`: both named FPs suppressed.

The same rule fired on the **true positive** in `soft-delete-filter`. Angle D found it, at the
right lines, with the right fix — and then wrote, in its own summary: *"Bugs found (confirmed): 0.
Hypotheses resting on unreadable code: 1."* The fixture's tree has no `base.py`, so the
soft-delete helper genuinely is unreadable, and the rule fired as designed.

So the trade this change makes is **precision bought with confidence**, not precision bought with
recall. FIND still locates the bug. What changed is how loudly it asserts it.

**Predicted downstream consequence (NOT measured).** The scorer caps at 50 any finding whose own
text admits it rests on unreadable code, and the threshold floor is 60. A finding that self-labels
as a hypothesis would therefore route to **advisory**, not the fix-loop. Under that prediction,
`soft-delete-filter`'s planted defect would be reported to the human rather than auto-fixed.
Nothing is dropped — but it is not fixed automatically either. **This is a design claim. The SCORE
stage was not run.**

Whether that is the right routing is a judgment call, and it is the one thing in this change worth
a human decision. Two readings, both defensible:

1. **Correct as designed.** The code the bug depends on is not readable. Auto-patching on a
   hypothesis is precisely what the cap exists to prevent, and a human sees it in Advisory anyway.
2. **Too blunt.** In a real repo `base.py` *is* readable, D would confirm rather than hypothesize,
   and the demotion would never occur — meaning this is an artifact of a context-starved fixture,
   not a real-world recall loss.

Reading 2 is plausible and I have **not verified it**. It would take a fixture with the model file
present, which this set does not have. Recorded as open, not as fine.

## Two angles this benchmark cannot measure

Stated plainly rather than left to be inferred from a table of zeros:

- **Angle C (cross-file tracer) is structurally inert here.** Every fixture is a single-file tree
  (`git ls-tree` confirms one tracked `.py` file). There are no callers to trace, so C cannot
  contribute and its zero-findings result on 4 of 5 fixtures means *nothing was there to find*,
  not *the angle works*.
- **Angle F (compound read-back) never ran.** It reads `docs/solutions/`, which the fixture repos
  deliberately do not contain — the same exclusion that makes the run blind also makes F a no-op.

**2 of the 6 angles are therefore UNMEASURED by this benchmark.** The 3/3 and the 0 FPs are
produced by A, B, D, and E alone. Any claim that "the six-angle finder is validated" would be
false; four of them are.

## Behavior worth recording (qualitative)

- **Angle B declined to invent work.** On `missing-await` (a pure addition — the diff deletes
  nothing) B returned zero findings and said why. A removed-behavior auditor that manufactures a
  removed behavior would be the obvious failure mode of this angle; it did not.
- **`excess-scope` stayed CLEAN, unanimously.** A, B, D, and E each explicitly considered and
  rejected the duplicate-dependency and layering complaints — the exact commentary that caused the
  altitude round-1 regression. The trigger gate is holding.
- **`intent-gap`'s two findings are both real, so neither is scored as an FP.** They are (1) the
  empty-name guard present on `POST` and absent on `PUT` — which is the fixture's *planted* defect,
  whose expected oracle is `/intent-review`, so this is lane bleed rather than a false claim; and
  (2) the `name.strip()` validated-but-unstripped-persisted bug, which is the fixture's **known
  unintended real defect** already recorded in the altitude run. Both A and E also flagged the new
  `repo.update(user_id=…)` kwarg as **unknown** rather than asserting a `TypeError` — the
  discipline working on the highest-impact-looking claim in the diff.

## Caveats & limitations (honesty rules)

1. **Fixture pre-images were reconstructed by hand** from each fixture's `diff.patch`, so line
   numbers differ from the canonical fixtures by a line or two (`missing-await` reports `:15` here
   vs `:17` in the altitude run — same statement, same fix). Defect identity was checked against
   `truth.md`, not line equality.
2. **n = 1 per fixture.** A single sample cannot separate a prompt improvement from run-to-run
   variance. The 3 → 0 FP result is large and mechanistically explained, which is *evidence*, not
   proof.
3. **The altitude run's 3 FPs were never attributed** to a cause. This change alters the finder
   again, so while the FPs are gone, the confound is not resolved — I cannot prove *this* change
   removed them rather than variance doing so.
4. **SCORE was not run.** See above. All routing claims are predictions.
5. **Scope of the claim.** This measures one skill's FIND stage against five seeded defect classes
   in single-file trees. Not the full chain, not real-world catch rate, not defect classes absent
   from the fixture set.
