## ⚠️ Harness caveat — read before trusting any number

This run used a **condensed harness**, not the full `/correctness-review` + `/intent-review`
skills. Each fixture got **one blind reviewer** (`reviewer` agent, model `sonnet`, blind to
`truth.md`) doing a combined correctness + intent pass and self-reporting a confidence 0–100 per
finding. That self-reported confidence is used here as a **proxy** for the SCORE stage's output,
so the threshold-75 analysis is "would a finding at this confidence enter the fix-loop", not a run
of the real 6-angle FIND → independent-scorer pipeline. Numbers are therefore indicative, not the
skill's production behavior. `not_observed != absent`. This is a **single run, n=7**, one model,
one Claude Code version — trend nothing off it alone (issue #61 protocol).

## Surprising finding (worth acting on)

The two new fixtures (`context-rule-unread`, `stale-inline-policy`) were designed as **expected
misses** for correctness/intent — the whole premise of the Phase-2 `/context-propagation-audit`.
In this run the blind **correctness** pass **caught both** (at 70 and 85). Why: each fixture
compresses the entire defect into a **single self-contained prompt-file diff**, so a capable
reviewer that knows about path-scoping can see "referenced but never Read" / "inline list is a
stale subset" directly in the diff.

The real PR #141 escapes were **not** so contained — the defect lived in the *relationship between
separate contexts* (an implementer dispatch prompt vs. the isolated subagent that consumes it;
a reviewer prompt vs. its plan-blind runtime context), which no single diff hunk exposes. So:

- The fixtures are **valid regression guards** (they lock in the specific escaped shapes) but are
  **easier than the escapes they model** — they overstate what correctness-review alone can catch.
- The `/context-propagation-audit`'s load-bearing value is the **cross-context** case, not the
  single-diff case. A future fixture revision should add a harder variant where the reference and
  the consumer live in *different* files/contexts so the defect is invisible to a single-diff
  correctness pass — that is the honest test of the audit's necessity.
- Recorded to `docs/review-escapes.md` follow-up and the fixture corpus, not silently smoothed over.

## Limitations

- Condensed harness (one blind `sonnet` reviewer/fixture), not the production `/correctness-review`
  6-angle + independent-scorer pipeline; confidences are self-reported proxies for SCORE.
- n = 7, single run, one model (`sonnet`), one Claude Code version (2.1.217). No variance estimate.
- Confidence-as-scorer-proxy is optimistic: the real scorer is a separate independent agent and may
  score differently; `soft-delete-filter`@60 and `context-rule-unread`@70 sitting just under 75 are
  exactly the cases where an independent scorer could land either side.
- Measures only whether these two skills catch these planted classes on these fixtures — nothing
  about real-world catch rate or unseeded defect classes (review-chain claim discipline).
- The threshold-75-vs-80 discussion is **inferential**, not a literal head-to-head: this exact
  7-fixture set was never re-run at threshold 80 (the older `results/*.md` measured a different
  fixture set), so the 80 column is reasoned from the same confidences, not independently observed.
  A true A/B needs the identical fixtures scored under both thresholds in one sitting.
